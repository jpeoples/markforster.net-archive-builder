New System to Eliminate Rewriting

Ok, while we wait for Mark's SF3 system, I thought I'd share a new system I'm thinking about doing as a test.

My hand tires easily if I write too much. So, this system is geared to keep from rewriting.

The basis of the system is AF1 with a few differences. Instead of writing on every line of the notebook, you write on every other line. The empty line below a task is used for dating when you worked on the task. Use 3 and 4 digit numbers for dates (day is always 2 digits): e.g., 101 is January 1 and 1001 is October 1.

So, if a task is completed with one effort, then there will only be one date on the line below it. If a task requires several efforts, then the line below will have a number of dates corresponding to when you worked on it, with the last date in the list being the last date you worked on it. If you were to work on a task so many times that there is no more room to write dates, then scratch through the task and rewrite it. On the line below the rewritten task, write the number of times it's been worked on, circle the number, and then put the date you rewrote the task.

This will serve to keep the list from artificially expanding. In addition, it will help to keep you from fooling yourself into thinking you've accomplished quite a bit, when in reality you've only crossed off a lot of insignificant items and moved a bunch of items with little-and-often. It gives you a better read on what you're accomplishing.

At the beginning of the day (or at the end of the day making ready for the next day) review the entire list of tasks. Flag as many or as few of the tasks that stand out by using Post-It Arrow Flags. This becomes your closed list to start the day.

Pulling the flag off of a task when you've done work on it gives you the psychological boost you need to keep going. Once you've finished the Flagged Tasks or none of the Flagged Tasks stand out any more, then you have completed work on your "closed list" and you then move through the list as you would in AF1.

Use AF1 dismissal rules – that is, when you cycle through the tasks on a page and nothing stands out, then highlight the tasks for dismissal. If the review of a task shows that the task is still valid, then rewrite at the end of the list. On the line below the rewritten task, write "D1" and circle it, write the number of times the task has been worked on, circle that number, then write date you rewrote task.

Using College-Ruled paper will allow you to put more tasks on a page than with regular-ruled paper. Or, you can use two columns on a page. In this case you would try to write your tasks in as few words as possible leaving room in the second column to write the dates.

I think I'll give this a try.

I guess if you didn't care about knowing the dates of when you worked on a task, you could always track the number of times you've worked on a project by marking a vertical line "|" every time you work on it. Every fifth time you work on a task you mark a "/" through the previous four "|" lines. This will easily show blocks of 5 times worked.

You can put this on the same line as the task. This might help those concerned with having half the tasks on a page they normally would have.

For pages with recurring tasks, (For example I have a sheet for gym exercises) I'm already following this scheme.

Using grid paper, I write the tasks down the left side of the sheet. Each time I work on something, I mark a square to the right of the item.

The second time I work on it, I mark the second square. Then to keep things aligned,all other tasks I do also get marked in the second square to the right of those (even if it's the first time for those) , so I know "when" I've done it.

The mark in each square can be a date, a check, a line, a block, or (for workouts) - the reps and poundage.
