2009-06-30 12:55
Claudia

1). you can set the default context for a project in the Project panel of the Inspector - the third panel in the Inspector.

My setup basically only has one project for autofocus purposes - the 'Autofocus' project for which I've set the default context to be 'active'.

2). I think the reason I don't see the jumping between grouped and ungrouped is that I am using Perspectives to jump between the project view and the context view and that Perspectives holds the grouping behavior unchanged. So I set up my desired context view, with ungrouped as the behavior and with the 'active' context highlighted, then take a snapshot of that window to save as a perspective.

Does that make sense?

When I first started playing with OF and AF2, I thought that the grouping and sorting in OF would let me down, but so far it has been rock solid, both on the desktop and the iPhone - my list is staying in exactly the same order I entered the items, in both project and context views. So it is possible, but I think Perspectives here are the key.

The other issue here is that I found in context view if I directly entered a new item, it would sort to the top of the list, whereas if I used the Quick Entry window, it would end up at the bottom of the list. So my default behavior is to use Quick Entry for adding tasks.

Again, I realise this probably makes it all sound complicated, but it's been a revelation how simple it has actually been.

Jupiter,

Out is simply for errands - shopping, banking etc. Typically stuff I do in town after dropping the kids at school, before heading back to my home office. I keep it off my autofocus list following Mark's suggestion to this effect. It just allows it to get synced to the iPhone so I have the errands list in my pocket.

The Out project and my Autofocus project are kept very separate in OF. There is no mingling of the items at all, and OF allows me to do this in a simple manner.

I've had a great couple of days with OF and AF2 - I love the ability to jump between the whole list (with active, completed & dismissed tasks all visible) to the 'active' context view, with just the available tasks to be worked on.

2009-06-30 22:19
Jupiter, to choose 'active' as the context in your perspective, set up the window just how you want it in context mode with 'active' selected in the sidebar, then choose "Save Window as New Perspective". You can then name the Perspective as you wish.

I've also found that if I add tasks on context mode, they go to the beginning of the list - so I only add in project mode OR by quick entry window.

Claudia, you do have to enter the 'autofocus' project in quick entry, but can do so quickly by tabbing to that field then typing 'af' as OF will locate the project based on the abbreviation. If you have set 'active' as the default context for your 'autofocus' project, then you can leave the context field blank - the task will automatically get the 'active' context.

Again, if you have set your preferences (in the main preferences window, under 'Data') to 'Cleanup Inbox Items which have a Context' and have set your 'Default List for Single Actions' to your 'autofocus' project then all should be good.

You should also set your autofocus project to be a 'parallel' project via the Inspector.

Claudia, you're correct about what you can see in context mode. So I have a Perspective that shows everything in Project mode which I have named 'all' - it's the closest approximation to a paper list if you were running AF2 in a notebook, in that everything is visible at all times. The settings from the Perspectives window are:

View Mode - Planning
Focus - No Focus
Filter- Remaining
Grouping - Ungrouped
Sorting - Unsorted
Action Filter - Any Status
Time Filter - Any Duration
Flag Filter - Any Flag State


Then I have an 'active' Perspective in context mode that doesn't show completed, only 'remaining':

View Mode - Context
Focus - No Focus
Filter - Remaining
Grouping - Ungrouped
Sorting - Project
Action Filter - Remaining
Time Filter - Any Duration
Flag Filter - Any Flag State

2009-07-1 12:02
Hi Claudia

The key to the Inbox issue is the preference setting in the main OF prefs: "Clean up Inbox items which have a context".

So as along as items have the 'active' context, they will be cleared out automatically. This is another reason to ensure that the 'autofocus' project automatically assigns the 'active' context to items. That way, you only need to send an item to the 'autofocus' project via the quick entry window and it will end up in the right place.

At this stage I'm not keeping a separate projects list, nor doing a weekly review. I'll see how this goes as time goes on.
