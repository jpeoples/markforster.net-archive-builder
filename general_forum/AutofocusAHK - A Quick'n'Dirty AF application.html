Hello everyone,

today I decided to write yet another Autofocus application. I wanted something that was quick, unobtrusive and portable. If you work a lot with keyboard shortcuts, this might be for you.

Here is Version 0.1: http://cloud.github.com/downloads/andreashofmann/AutofocusAHK/AutofocusAHK.exe

Just download it and save it anywhere on your hard drive. I can run from usb drives, too. All tasks will be saved in a text file in the same directory, so it's 100% portable and leaves no traces elsewhere.

Okay, here's how to use it. It's probably different from anything you used so far for Autofocus. ^_^

Start the application, and you get a tray icon that says "af". Right-clicking this gives you a very small menu: About/Help and Exit. About/Help shows an about box with all keyboard shortcuts, Exit closes the application.

Here are the shortcuts:

CapsLock + a : Pops up a window where you can enter a new task, which will be added to the end of the list

CapsLock + d : Shows the current task and asks you if it feels ready to be done. Click yes to start working on it. Click no to go to the next task. Clicking cancel or pressing Esc just closes the window again.
If you've worked on something for a while, press CapsLock+d again and it will tell you what you've been working on and whether you want to re-add the task to the end of the list. Yes re-adds, no doesn't, cancel let's you work on the task a little longer.

CapsLock + c: Shows the current task.

CapsLock + s: Shows the next 30 tasks

Note that I use CapsLock for Hotkeys, because I don't have any use for it. If you use CapsLock for anything, you currently can only change the hotkeys in the source and compile it yourself.

The source can be found here: http://github.com/andreashofmann/AutofocusAHK/tree/master

Note that version 0.1 lacks forward mode and review mode so far, all you can do is add, do, re-add, and move through the list in reverse mode. But when the next version is ready (tomorrow), you can just copy it over the old one and continue with the same list.

Here is the file format of Tasks.txt:

Every line looks like this:
{Task description}{Tab}{Metadata}

Task description is what is shown to you in the application. I can contain anything except tabs.

Metadata contains mainly timestamps. Each timestamp has a preceding letter that indicates what it represents.

Example:
Implement review mode A20090821100628

The A stands for "added". When the task is done, a new timestamp is added with a "D". When a task is put on notice, it's marked with "N" (but without a timestamp). Once it's on review, it gets a timestamp with "R". The timestamp in the metadata are separated by spaces, there is only one tab in the line, the one between description and metadata.

So this is the life cycle of a task:

1) Axxx -> Added
2) Axxx Dyyy -> Done before being put on notice
3) Axxx N -> On notice
4) Axxx N Dyyy -> Done after being put on notice
5) Axxx N Ryyy -> Put on review
6) Axxx N Ryyy Dzzz -> Re-added in review mode

2009-08-21 15:56
Thank you.

The master branch now contains the code for the morning routine. Every day a user-defined time, tasks that are on notice and unactioned are dismissed, then the next block of unactioned tasks is put on notice. When AutofocusAHK is started for the first time, it creates a configuration file, which looks like this so far:

[ReviewMode]
LastRoutine=20090821
StartRoutineAt=6

LastRoutine is the date the last routine ran, this value is updated automatically every day. StartRoutineAt is the time at which you want the morning routine to take place. It can be a value from 0 to 23. Default is 6 in the morning. Depending on when your day starts, you may want to change this. If the app is not running at that time, the morning routine will run immediately when it's started.

Okay, I discovered a potential issue with Vista and 7 - If the window that has focus has elevated privileges (runs as Administrator), the AutofocusAHK shortcuts won't work. There is not much I can do about it.

One solution is to run AutofocusAHK as Administrator by rightclicking the exe file and selecting "Run As Administrator". But that doesn't fix Autostart.

Another option would be to disable UAC, which would fix autostart, but I don't recommend it.

The pragmatic solution is to accept that the shortcuts don't work if you're in a window with admin rights. Just click once onto the taskbar and the shortcut will work again.

Normally I don't like applications that need admin rights, but I don't think it's possible so have global hotkeys that work on elevated rights windows if the don't have elevated rights themselves.

Oops, my mistake. That means you're in the open list. When you're in the closed list, the title says "Pass x - Closed List - AF4". Seems I replaced only the "Pass x" part with "Open List" instead of "Pass x - Closed List". I'll fix it. Btw, when you haven't taken any action on a pass of the closed list, the tilte says "Pass x(!)" to remind you that the closed list will be dismissed if you don't do something.

"When do tasks get into Closed List from Open List? According to the original instruction, if no items stands out in the Closed List, the tasks will get highlighed (on-notice status) and Open List items will become Closed List items. However, this didn't happen when I began with a new task list and keep clicking 'no.'"

The thing is, at the beginning there is no closed list, only an open list. That's why no dismissal happened for you when you kept clicking no, because the open list does not get dismissed. At some point, the first open list must be declared closed. The instructions are not specific as to when exactly that should happen. I decided to do it when the list contained enough items to fill a "page". So it depends on what ItemsPerPage in the ini file is set to (default is 15). When you start the application and you have more items than this, the list becomes closed.

Perhaps I should add a trigger to the "add task" routine so it can happen without restarting the application in case someone never restarts it (computer always on or only in stand-by).

Steve:

The standard shortcuts are:

CapsLock + a: Add a task
CapsLock + s: Show the next tasks and tasks on notice for review
CapsLock + d: Start/Stop Working
CapsLock + e: Export list to HTML
CapsLock + p: Show preferences
CapsLock + r: Reload the tasks list from Tasks.txt
CapsLock + q: Quit the application

You can also find these in the README.txt file and in the about box (double click the tray icon or right-click and select "About/Help".

I used CapsLock as a modifier for hotkeys because not many people use it (at least not intentionally). ALL CAPS TEXT SUCKS ;-)

Unfortunately, using CapsLock in its normal function is not possible when it's used as a modifier key. But I changed release 0.9.2.2 so that CapsLock doesn't get blocked if you don't use it as a modifier.

To change the hotkey, open the file Autofocus.ini, which contains this section:

[HotKeys]
HKAddTask=CapsLock & a
HKWork=CapsLock & d
HKShowNextTasks=CapsLock & s
HKExport=CapsLock & e
HKPreferences=CapsLock & p
HKReload=CapsLock & r
HKQuit=CapsLock & q

You can assign your own hotkeys here. If no hotkey uses CapsLock, it will function as usual. The rules for defining hotkeys can be found here: http://www.autohotkey.com/docs/Hotkeys.htm and in the README.txt file.

For example, the line

HKAddTask=^+a

would mean that you add tasks by pressing Str+Shift+a instead of CapsLock+a.

If you want the default hotkey back, you can just delete the whole hotkeys section from the ini file. You need to reload the application to apply changes in the ini file.

Reply by Andreas Hofmann

2010-02-1 14:48
Rebecca:

The program is pretty much just the one exe file and some text files in the same directory, so if you removed that, the program is gone completely. What you still have is a link in your windows startup folder to my application, generated when you checked "Start AutofocusAHK with windows" in the preferences. So the error message you get is because windows wants to start my application, but can't find it because you deleted it.

To get rid of the error messages, just open your start menu, go to "All Programs" (or just "Programs", depending on your windows version) and find the folder "Startup". There should be a shortcut called "AutofocusAHK". Right-click it and select delete. That should be it.
