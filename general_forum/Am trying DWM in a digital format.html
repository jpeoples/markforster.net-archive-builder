I think that it is fine to try new things, but we need to be aware that time management systems are like an investment. They require an investment of time to learn before they pay dividends in time saved. They also require an ongoing investment of time to maintain. It is sometimes hard to quantify the return on the investment. In theory if you spent no time on time management, 100% of your time would be available to actually do things. However, for me, without some system in place, I would forget things and it would reduce my effectiveness and cost me time. So I look for the simplest way to make sure I don't forget things. Obviously everyone has their issues and should make sure their system addresses them.

Additionally, as Mark has mentioned, if you are over committed no time management will solve this.

I appreciate the laboratory like quality of this forum since it is with experiment that we will all find what works for us.

I did the following trick to get some visual context of the next 7 days and 4 weeks for my digital implementation.

For the upcoming next 7 days add "-" as a daily delimiter with the respective due date set. If you use DWM in the office and don't want to use Sa/So only add the delimiter for the next 5 workdays. To get the delimters sorted in after all the real tasks of the respective day set the priority to "-1 Negative" if you use Toodledo (sort by due date, priority). If you use MLO set the time part in the due date to 11:58 pm instead.

For the upcoming next 4 weeks add ".========== week ============" as a weekly delimiter with the due date set at the end of the respective week. In the office Friday is my end of the week. At home I set it to Sunday.
Again, to get the weekly delimters sorted in after all the real tasks of the respective week set the priority to "-1 Negative" if you use Toodledo. If you use MLO set the time part in the due date to 11:59 pm instead (1 Min. more than the daily delimiters).


When today is over just forward the due date of todays "-" by one week so it helps to divide the updcoming week. Thus with each passed weekday you get one more delimiter visible in next weeks time frame. If the week is over forward the due date of this week's delimiter by one month.


Pros:
+ You get a pretty good visual representation of the due tasks during each of the next 7 days and during the next 4 weeks.
+ You see days or maybe even a week where no task are due (this is a real fine feeling).


Cons:
- Your number of tasks is not really correct any longer, as the delimiters are counted as tasks as well.

If you are using Toodledo and want to see all your tasks listed day by day, for the coming month, as you would in a page-a-day diary, try using the calendar view.
