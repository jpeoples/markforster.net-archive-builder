You came to a page, and found nothing on that page worth doing. It should be dismissed. Writing a new task should not save that page from destruction. Dismiss the page. (Remember: you can still save dismissed tasks by rewriting them.)

Vegheadjones,

I believe that, according to my consolidated instructions, and Mark's revised rules, you should just proceed to the next page without dismissing, since you did one task. If you come all the way around to the same page, and find nothing actionable, then it is time to dismiss all the remaining tasks on that page.

But I see what you are thinking about, and I believe this scenario is what Mark was trying to avoid when he had the somewhat-more-complicated rules earlier. If you want to game the system to avoid never dismissing a page, you can always think of one "urgent" thing to add and do it, then jump to the next page, while never clearing the "backlog" items.

Reply by Mark Forster

2010-12-1 11:07
Yes, ubi is correct. Under the initial rules you would have dismissed the page, but under the revised rules you do not.

The reason I changed it was because when Column 2 fills up completely no new tasks can be added to the page anyway - so it wouldn't be possible then to continue to game the system by adding new easy tasks.
