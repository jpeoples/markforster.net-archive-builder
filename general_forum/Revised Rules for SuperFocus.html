I don't think the original rules were as clear as they should have been, especially Rule 4, so here is a revised version which I hope will be more understandable. Please note this is purely a matter of describing the system better - it does not involve any changes to the system.

------------------------------------

Normal AF1 rules, except:

1. There is a second column (Column 2) on each page which can be used when you are working on a page for:

a. Re-entering unfinished tasks;

b. Entering tasks which you want to do now (Note 1)

2. Column 2 is treated as a part of the page for all purposes, including dismissal.

3. If there is insufficient room in Column 2 for more tasks, then any further tasks for it are entered in the 2nd column of the next available page. (Note 2)

4. When you have taken action on a task and have re-entered it in Column 2, but there are no other tasks remaining on the page, you may transfer the task to Column 2 of the next available page. You may also transfer the task in the same way when there are other tasks remaining on the page but you do not wish to take action on them for the time being. (See Note 3)

5. If under Rule 4 a task is transferred to another page but no other task stands out for action on that page, then the transferred task is transferred again and all tasks remaining on the page are dismissed (including any in Column 2). (See Note 4)

6. The purpose of rules 4 and 5 is to prevent a re-entered unfinished task from not having any other task to alternate with.


Notes:

1) This is intended to be used for anything you want to do now, not just urgent tasks.

2) This imposes a limit on how long new material can be added to a page.

3) The effect of this is to allow one task to continue to be worked on for more than one page.

4) This means a page does not avoid dismissal just because a new task has been added to it under Rule 4.

Well, the main disadvantage would be the difficulty in keeping track of which pages are active and which are not. If you've resuscitated a page which is a long way back in your notebook, you may keep forgetting about it.

I expect it's quite possible to think up some way of overcoming this, but why expend the mental energy when all you have to do is write the task at the end of the list? Re-writing the task also helps to recommit yourself to it in your mind.

2010-11-15 22:16
In AF1, every page but the last is full. Here, the left column is always full except for that of the last page, which is open to new tasks.

You aren't usually working on that last page, so whatever you work on must go to the right. It won't fit anywhere else.

When you *are* working on the last page, you would still rewrite to the right so a suitable variety of new tasks can fill up the left side.

A new task always goes at the *end* of the list in the left-hand column, unless you need to do it now. In that case it can instead be added to the page you are currently working on. It has to go in the right column because the left column of every page (except the last page) is full.
