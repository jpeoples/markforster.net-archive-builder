I've found myself using AF1, AF4, DWM, and SAF in multiple combined fashions, applied as techniques for managing certain areas of my life and work, but not using any one of them as an overarching system.

It has been very effective.

For my overarching system, I use Gerry's 80/20 rule: write down everything in one place, and keep a file system using some kind of simple organization. Those two items account for about 80% of total TM success. At least that's what Gerry says, and I think he makes a pretty strong case.

Sometimes I process my main list with SAF rules, sometimes with AF1 rules, sometimes with no rules at all except the "write it all down in one place" rule. Whatever seems needed at the time.


And then I use AF and DWM lists for managing all kinds of categories of things:

- DWM list for managing project tasks of different kinds

- DWM list for managing miscellaneous email reading

- AF method (slightly modified) for managing books on my shelves that I want to read

- Standing out, dismissal, "just do something", etc. -- using these principles all the time


Anyone else doing something like this?

My method is based on DIT, AF and DWM combined.

The core of my daily system is a datebook with planned tasks, like DIT before Mark suggested we not write tasks too far in advance. It includes tasks that are repeated, tasks that are one-time-only, and steps in bigger projects that have deadlines. When I finish a task that is to be repeated, I write it on the next due date. I plan projects backwards from the deadline and include a good deal of extra time. As I check off each step, I confirm that the next step is planned.

Every time I work a list, I use the AF method of looking through it and accepting that something will jump out at me. If nothing jumps out, I do the first on the list.

If I'm up to date, I treat today's list and the days around it together as one list, picking the task that jumps out. I might also go to my someday/maybe list, which I treat as an AF1 list.

If I'm behind, I start the day by reviewing all outstanding tasks and making a list of things that should be done in the next few days, then cycle through that list doing things as they jump out. Since I've already said everything on that list must be done in the next few days, I stay on that list until it's done.

Then I go to the oldest day with unfinished tasks. I cycle through that list until it's done, and celebrate when I cross of the day. I tried treating the book like an AF1 list, with each day being one page, but found that finishing one day at a time worked better.

If a task is blocking me and causing me to procrastinate, or I realize it should wait until I catch up, I move it to a new date. When I write it on the new date, I include a note of the old date. When I reach the new date I see how long the task has been on the list. Then I apply DWM, but with a bit of flexibility about the date. If I've had a truly busy month, I let it go a bit longer. Eventually, though, I have to think carefully about why I want to do it and why it's not getting done.




Then I go to the oldest day with unfinished tasks and pick tasks as in AF1, except I don't get to move off that page before it's done without good reason.

I let myself carry tasks forward only if I realize the task is blocking me. Sometimes I will do anything but the task in question.
