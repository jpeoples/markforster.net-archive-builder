I have now posted a complete re-write of the instructions on my blog. In order to make a coherent whole I have extensively revised the nomenclature.

The system is now called "The Revised Autofocus System". I intend to quietly drop the "Revised" bit after a while, so you can continue to call it Autofocus or AF.

AF1 is now called "Forward Mode"
AF2 is now called "Reverse Mode"
Dismissal is now called "Review Mode". Task are put on notice for review, not dismissal.

I've also got rid of the expression "standing out" which many people found hard to understand and didn't translate easily into other languages. I now talk about tasks "feeling ready to be done".

I've also included a pass rule for Forward Mode (AF1) so that you can continue in Forward Mode if your location prevents you from doing any tasks on a particular page.

The basic thing to keep in mind is that AF works best for "discretionary time". How you define that can be tricky, it turns out, but that is the basic thing to keep in mind. For example, if you are a factory worker and you stand at a moving belt and inspect widgets as they fly past ... you are NOT in discretionary time. You are told what to do and you do it. No decisions of any magnitude to make there.

If you have to file a monthly report to your boss, the "task" is not discretionary, but how and (exactly) when you do it are. See the difference?

What makes this just a bit tricky is that we can define some of our own tasks as not discretionary, if we work best that way. Just thinking in the shower (I don't sing ;-) The principle behind testing list membership is that the list has a PURPOSE. For example, if you write a list of candidates for gifts to get someone for a birthday, you don't put on the list "Water the dog", "Feed the Cat", "Buy Tomatoes". Why?
