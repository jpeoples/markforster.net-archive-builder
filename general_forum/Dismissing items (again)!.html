Reply by Christine B

2009-01-27 23:42
The easiest way to deal with items such as these is to rephrase them. The process of rephrasing effectively analyses the type of task and identifies whether a further system is needed to control that type of task in the future.

For example, if I have on my list a task "buy sausages" which I cannot do until I go to the Supermarket, I change that to "add sausages to shopping list", create a shopping list and use that for all future shopping list items.

If I have something that is dependent upon something else, e.g. "email Fred with report on xyz" which requires the report to be completed before it can be actioned, I would treat it like a project and action it by replacing it with the "project" steps, e.g. obtain data for report, compile report, email report to Fred.

I am finding that the more I work with AF the easier it is to recognise which items can truly be dismissed and which need to be rephrased or handled differently than originally anticipated. I think that type of item will become rarer as we will be training ourselves to write our tasks in a different format going forward.

Hope that makes some sense ...... :-)
