if you still want to use Palm Desktop, you might be interested in my latest post concerning using a Palm for AF:
http://www.markforster.net/forum/post/673409

Since Palm Desktop mimicks the functionality of the Palm handheld, you could use the same setup in Palm Desktop. However, it can't filter by priority (AFAIK), only by category, but that will suffice if you move dismissed items to a 'Someday' (or 'Dismissed') category (unless you want to delete them right away, of course). You can then regularly filter out that category and only view (only) that category to review the dismissed items.

I did a test today and found that you can do everything you want with Tasks on the Palm.

- Forget priority as it defaults to 1 and it would be a pain to change it each entry
- Use one Category for your main list and another for Someday and ... well, whatever
- Tick it off and either leave it there or have it hide
- Use notes for extensive additional information
- Put in a page marker every 30 entries or so
- Add a date when you think appropriate and it sorts to the top ... pretty nice.
- Set an alarm if you like.

One small hitch is in the transfer from Desktop (where it is easy to do some power typing) to the Palm some (a very small percentage of my test data) of the items picked up a date, yet showed no date when I tried to change it. I had to change the date to a different date then change it to no date ... then they went to the end of the list. HUH? Well, no real problem ... it looks as if I worked on those items -- I got a freebee ;-)

It looks as though it might be a fiddly system given that the support for Palm is almost non existent.

Of course, the downside is still that it is slower to get out the Palm, power it on, write in the entry, etc. It is faster to whip out the notepad ;-) And there is still the risk of failure unless everything is backed up.
