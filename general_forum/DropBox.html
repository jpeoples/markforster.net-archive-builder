I just recently switched from Dropbox to Live Mesh from Microsoft. My primary reasons were:

- Live Mesh gives 5GB cloud storage, to Dropbox's 2GB
- Live Mesh has a remote desktop feature
- Live Mesh can sync folders/files between 2 computers only, skipping the cloud (handy to keep your 16GB music collection synced between 2 computers w/o consuming your 5GB web storage)

So I sync my documents between 2 PCs and the cloud, but my MP3s only between the 2 computers.

However, Live Mesh does not do file versioning like Dropbox does, so once you make a change to a file and it syncs to the cloud, that's the only version you have. I have other backup solutions in place to mitigate this risk, though.

Just an alternative to consider. Dropbox is a great service that I happily used until I decided my music collection must follow me wherever I go.

Chris:

A workaround is the use of junction points in Windows and symbolic links in Linux and Mac OS. It's kind of like a shortcut, but at a deeper level in the file system. For applications, it looks like the same folder exists in two places, but the space is only used once on the hard drive.

So if were on Windows and had your dropbox folder in C:\Users\Username\My Dropbox, you could create for example junctions points

from C:\Users\Username\My Dropbox\Photos to C:\Users\Username\Pictures\

or

from C:\Users\Username\My Dropbox\Projects to D:\Projects.

Then, everything you put in C:\Users\Username\Pictures\ or D:\Projects gets synced by Dropbox.
