As I haven’t seen anyone else do so (though I might have missed something), I wanted to mention both Things and The Hit List options for implementing DWM electronically. From what I’ve been able to figure out, there’s no way of implementing DWM in OmniFocus so that it runs and smoothly and as easily in either Things or The Hit List. The Hit List is best adapted, it seems to me, mainly because there’s less tabbing involved to add dates to tasks than in Things. Things wins out in two other ways, though: Things handles scheduled tasks (i.e., tasks you don’t want on the list yet, but want to be reminded of at some future point – like my driving licence renewal in a couple of years, say) and repeating tasks in a nicer way than THL, and, notoriously, THL has no iPhone app. (The developer has been claiming that it’s on the way for a long time; there are rather heated debates out there about what to expect, etc.) Not only does Things have a very nice iPhone app, but it’s relatively free of sync problems, unlike OmniFocus’s iPhone app.

Hi Alan:

Yes, Put Things Off is very nice as far as it goes. It does (quite deliberately) do a lot less, though. It’s iPhone only, for one thing (at the moment). It doesn’t handle repeating tasks, or have a special place for notes. And, perhaps most signficantly for DWM, it’s a much more cumbersome process than in THL or (the desktop version of) Things to assign dates in PTO. (You can set a default amount of time to put things off by in PTO, but in DWM one is going to be constantly switching between dating things for a week from now, and a month from now, and switching from one to the other in PTO is not quick.) Of course, any of this could change when the developer brings out PTO 2.0.

Haven't had a single problem with Omnifocus syncing, so I'm not sure what Martin is referring to.

My DWM set up is as simple as can be using Omnifocus - again, I don't understand what the problem is.

Here are the basic elements of my set up:

- I use a single context for all my active tasks

- This context is set up to show only active tasks

- Tasks are automatically sorted by due (expiry) date

- Expired tasks are moved (dumped) to a project labelled "Expired". (NB. this project's status is set to "Dropped" so that these tasks don't appear in the active task list.

- I follow Mark's instructions or cycling through the list.

Not sure how Things does this any more elegantly.
