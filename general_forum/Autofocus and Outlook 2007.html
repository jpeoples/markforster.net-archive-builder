2009-01-26 0:19
Just checked Outlook 2007 - you should be able to set up the filters I suggested.

If you go to View>Current View>Define View. Then choose "New" - name your view, keep it on "Table" (default), click okay - it will give you some options to create filtered views.

To create a filter to see categories with "page" in it:
Click Filter, Click Advanced. Click Field, Choose "frequently used fields", click Categories. Choose "contains" (default) and enter "page" into the blank field.

To create a filter to see tasks with no "page" category attached to it:
Click Filter, Click Advanced. Click Field, Choose "frequently used fields", click Categories. Choose "doesn't contain" (default) and enter "page" into the blank field.

You probably should also add the creation date field to these views and have it sorted by it. Back in the customisation options for your view (you should be returned to this after setting up the filter but if not, you can get to it later by going to View>Current View>Define View - select the view you want and click modify).

Add a creation date to your view.
Click on Fields. You should see in the drop down list "Date/Time Fields", select this. Then you should have a field called "Created" - this is your creation date field. Select it and click on "Add->" to add it to your list of fields. You can drag it up and down in the list to change the order.

Automatically sort by creation date (if you do this then you don't need to group by Pages)
Click on Sort. At the bottom of the popup, you'll see a drop-down "Select available fields from" - choose the "Date/Time Fields". Back at the top of the popup, you should now be able to select "Created" for "Sort Items By". I think it's "Ascending" to sort them with the oldest showing first

2009-02-22 20:59
While start dates do not have specific times, the creation date has a time attached. So if you use start dates (and you always set them to "today"), you can group by them, and in addition sort by creation date, so the order of the items under one 'page' (= one group) will always stay the same.

IMO, grouping (or a combination of grouping and sorting) is better than simple sorting. You don't need dummy 'page break' tasks, and you can expand one 'page' only, while collapsing all the others. This gives you a better 'closed list' effect.

Grouping can be accessed by a right click on the headers of the task view, choosing something like "define current view" (unfortunately, I don't know how exactly it's called in an English version of Outlook, I have a German version of OL 2003) at the end of the context menu, and then using the 'group' button. The same goes for sorting. The sort order will only work inside of the groups. Creation date will only be accessible when you set it to show all date/time fields. And while you're at it, you can filter out done items.

OTOH, this also shows that the solution I proposed earlier does *not* work, unfortunately. Outlook builds a group for every creation date *and* for every creation time. So you'll end up having lots of groups of single items.
