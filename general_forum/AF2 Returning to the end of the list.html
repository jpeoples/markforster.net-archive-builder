Original Post

2009-06-29 12:48
With the release of AF2 I took the leap and declared a backlog and started a new list. I started with 7 tasks on my new list, which is now almost 3 pages long.

I understand the reason behind returning to the end of the list after completing a task, but I'm struggling with it and I wondered if anyone else was having the same problem.

One of the features I liked about DIT and AF1 was the gap between identifying a task and working on it because it enabled me to think about it and decide if I really wanted to do it. However with AF2 I could return to the end of the list and find a task I added a few moments previously which is likely to 'stand out' because I've only just added it.

I want to implement AF2 with no tweaks (at least to start with ;-) ), but as I've been working on my list, I have found myself wanting to continue through the list in AF1 style. Then when I make myself return to start I'm telling myself to ignore the last few tasks I've added so I don't end up just working on new tasks.

This doesn't feel right, so have I missed something in the instructions?

2009-06-29 12:53
What I've begun to find is that as the day goes on the tasks earlier in the list begin to loom larger in my mind and exert more of a pull, and I become more urgently aware of the tasks that are on notice of dismissal. What is happening as a result is that I am now skimming over the tasks at the end of the list and it is tasks earlier on that are standing out. I think this is why Mark stressed the need to read through the whole list at the start of the day, and how the system works to achieve a balance between older and newer tasks over the course of the whole day. That's how it is turning out for me anyway.
