The rules say that a page is "closed" if column 2 is full. For a page like that, the rules are very clear:

<<< 2. When you visit a page which is full (i.e. has no space left in Column 2), all tasks on that page must be either actioned or dismissed. Dismissal only applies to tasks in Column 1. Tasks in Column 2 must always be actioned. >>>

Since that page will imminently disappear, I don't see why it's a pressing problem.

It only becomes a pressing problem if a quickly-filling Column 2 is a chronic experience. But in my experience so far, this is a situation where SAFv3 really shines.

Overloading column 2 is a painful experience. It forces you to deal with all those tasks you've loaded in column 2, while column 1 items on future pages get neglected, and the system overall starts to really slow down. But the pain of this experience is actually valuable.

For example, I am like Matt, in that I have lots of unfinished work, and lots of little urgent things that come up during the day (some of them recurring Outlook reminders). This tends to overload column 2.

So, I am learning to do the following:

(1) If the system is forcing me to work on long-term projects at the expense of more pressing work, because I keep re-entering that unfinished long-term project in Column 2, then I need to break down that long-term project into smaller chunks, and enter those chunks into the system, rather than the whole project. (I can do this mentally, or literally by the way I write the task). This raises attention on that particular project and makes me think through it better -- what really needs to be done, and when, and why.

(2) If my recurring urgent tasks are filling up column 2, I can revisit those tasks and see if they are really as urgent and as necessary as I think they are. It basically forces an audit of these recurring tasks -- and like the AF4rev2? experience showed, these recurring tasks can sometimes make up the bulk of the items on our lists.

(3) If I am STILL getting too many unfinished things in column 2, this seems to be a sign that I have taken on too much work, and need to scale back my commitments. (Column 2 seems to be the gauge for measuring overload - a feature that AF1 was lacking. DWM was also good at gauging overload, but the SAF column 2 gives much faster feedback!)

In the past few days, I have already learned to treat Column 2 with respect, otherwise it will bite me. In other words, the pain of dealing with a chronically overloaded Column 2 is already teaching me to say "NO" more effectively. DWM wasn't really able to do that...

So, yes, an overloaded column 2 is a problem, but if you just keep following the rules, it seems like the problem takes care of itself and teaches better habits in the process.

Will:

<< Is this right? I don't see any limit in the system to the number of pages you can fill with column 2 tasks.>>

This is something which hasn't been spelled out properly in the new consolidated rules, though it was in the original ones. What is supposed to happen is that if you are on the last page, unfinished tasks are re-entered in Column 2 of the earliest active page of the list. In other words, when you reach the end of the list you go back to the beginning.

When there is no more room in Column 2 of a page for more tasks, then the tasks go into the first subsequent page which has room. If it's the last page which has no room, then the tasks will be entered on the first page using the same principle - when you reach the end of the list you go back to the beginning.

So you can see that if someone grossly overloads Column 2 with urgent tasks, it is in fact possible to fill all the Column 2s in the list.

I will shortly be doing an amendment to clarify this.
