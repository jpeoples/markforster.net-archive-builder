I still mix them in. For example, I was still using these with "AutoDIT".

Recently, I've changed it a little. When I come across a project page, I try very hard to find all the "next items", and then choose the main ones that stand out and put them on the main list.

This works pretty well and I can get a lot done that way.

But sometimes there are too many scattered items and I start to feel lost. For example, I get 12 emails from coworkers related to the project, and that changes everything I need to do related to the project. I need to pull it all together again and re-evaluate.

If that happens, I move each scattered task back to its project page, and reinsert the project page into the intermixed AF1 sequence. When I come to the project page during the course of processing the list, I sort through it all, figure out the "next items", etc., just like I wrote at the beginning of this comment.
