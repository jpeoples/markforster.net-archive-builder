Seraphim:

My view is that the system has already got that self-correcting function in it. But in order to get that self-correction to work, you have to stick to the rules as written.

The self-correcting mechanism is Column 2.

So to take the example you quote of having to many unactioned items and too many pages so you can't work through it fast enough - what happens?

Because it's taking you so long to get through the list, you will need to increase the number of urgent tasks in Column 2. This will speed up the process of filling up Column 2.

When Column 2 is full, you have to take some action on every remaining task in Column 1 or dismiss it. Either way the page is now no longer active.

Faced with the pressure to action or dismiss all the tasks in Column 1, you may well take partial action on many of the Column 1 tasks, so again the process of filling up Column 2 is accelerated (this time with unfinished tasks).

Whenever you succeed in filling Column 2 of a page there is a knock-on effect on subsequent pages because tasks can no longer be entered on it - increasing the process of filling Column 2 yet again.

So far from there not being a self-correcting action, there is in fact one which increases rapidly after a certain point is reached.
