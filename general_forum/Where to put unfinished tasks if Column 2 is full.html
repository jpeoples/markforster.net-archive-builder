Where to put unfinished tasks if Column 2 is full?

Author: Seraphim | Created: 2011-02-10 23:36 | Last Activity: 2011-02-12 3:21

Original Post

2011-02-10 23:36
Let's say I am working on Page 1. I stop working on a task. It is unfinished. So, I need to re-enter it on page 2, column 2.

But, page 2, column 2 is already full.


This is a problematic situation, obviously, and is a likely indication that I have been abusing Column 2. But in the meantime, while I learn to treat Column 2 with more respect, I need to know where to write things.

Perhaps a rule addendum could be made:

# if the task is not finished, delete it in the same way. Re-enter it in Column 2 on the next page. If Column 2 on that page is already full, then you must rewrite the task in such a way as to allow you to enter it on Column 1 of the last active page.


Or alternately (with a little more concession to the Column 2 abuser):

# if the task is not finished, delete it in the same way. Re-enter it in Column 2 on the next page. If Column 2 on that page is already full, then re-enter it in Column 2 of the next page that still has room in Column 2. If NO pages have room in Column 2, you must rewrite the task in such a way as to allow you to enter it on Column 1 of the last active page.


In other words, you really must reconsider the way you are breaking down your projects and how many tasks you allow to be called "urgent". These rule addenda force you to do that, if you are a Column 2 abuser.
