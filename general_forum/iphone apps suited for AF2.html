Thought this might help some people.

For another project we literally had to download every todo and task application for the iphone and review it. As part of that scan I kept my eyes open for ones that would be well suited to doing AF2 the why they are written today. Here's the list of those that survived out of 70 we looked at.

1) Toodeldo (can be customize to only use status, tar, repeat) and is then perfectly suitable for autofocus. Plus has web application for desktop

2) WhatNext. (fi you stay off the flash card screen, it's pretty much autofocus) no desktop application

3) Smart Todo (very basic, you'd have to delete to dismiss) no desktop application

4) Zenbe nice basic lists with a web application

5) CarbonFin outliner. mention in other discussions. no desktop application.

runners up...

6) Crossout - almost perfect but buggy in OS 3.0

7) TaskPro - overly complex but can do it. no desktop

8) Listomni - some like it; feels too kitchen sync to me.

9) Omnifocus - can manage AF2 but many many more features than needed and high priced; if you need the power it's probably got it.


So if you need a desktop access look at:

Toodeldo, Zenbe, and OmniFocus.

If you don't need desktop also look at

WhatNext, Smart Todo and carbonfin.

I've been using Things (a Mac desktop + iphone app) for a while and like it a lot. I stumbled on Mark's autofocus system yesterday only and am eager to try v2 of it.

I wondered if I could use this system along with Things, and seems the app will easily adapt:

- I will live in the Today screen, which will be the AF2 list.

- new items will get entered there, from top to bottom...

- ... unless I enter them only for future reference, someday projects, etc, in which case they will be put in their own context/project list.

- item that has been actioned but needs to be acted upon again can be duplicated, so one instance is crossed off and the other is moved back at the top of the list.

- crossed off items can be set to remain there indefinitely.

- I will create a dummy task with "--------------------" to act as a separator for potentially dismissed items. I then simply move that line at the beginning of the day.

- All unactioned items that fall beneath this line will be removed the next day by selecting them, marking them as "not Today" and adding to them a "dismissed" tag. That way they will be removed from view and sent back to their own context/project list, if any, or in the "Next action" list if they aren't attached to any context or project. It will then be easy to eventually review them. Since they will bear the "dismissed" tag, it'll be easy to differentiate them from other tasks/ideas that haven't been dismissed yet.

- If I want, I can add to any new action a context-sensitve tag -- so that if I'm going through the list at the office, I can choose to remove from view all those that were tagged as "errands" for example.

- I can also use Project lists and Areas list (like Personal, Work, Family, etc) whenever I want to brainstorm/plan/envision... and then browse through these lists once in a while, and mark either a whole project as "Today" -- in which case the first item in this project will be put in my AF2 list -- or a simple task.

All in all, that seems an elegant and simple way to start testing AF2 for myself. What do you think? Any drawback to using an app like Things?
