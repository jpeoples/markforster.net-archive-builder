Here's a sample image of my current setup.

http://dl.dropbox.com/u/394006/DWM_sample_image.jpg

The only way I have made DWM work to my satisfaction is to use an electronic approach. The program is ListPro. The file is stored on a microSD card. I use the same file whether away (on my smartphone) or at the PC.

The columns to the right are for filtering. Each task can (and usually does) have more than one filter.

The date highlighted magenta is today, and includes indented tasks which will expire today.

Then there are several yellow days, with green signifying a week from now, where tasks are moved to (drag and drop) after being worked on.

New tasks are entered at the end (26 April).

Recoloring and adjustment to the list is a quick morning operation every day.

The blue Bookmark line is dragged round and round the list (top to bottom), pausing when it is above a task which stands out. There is only one bookmark, that is why all the filters are set on it.

Other than ListPro, the only program I am aware of that enables this solution is TodoPaper (or the very similar TaskPaper). However these have no equivalent for Windows Mobile.

Todopaper is really like a simple text program. You just type and hit enter. After you create a new file, you just start typing. If you get rid of the leading dash and end a word or sentence with a colon, the entry is made bold (called a project). If you just hit enter, you will get a task entry (with a dash). If you take away the leading dash and just type, the entry becomes a note. You can use the tab key to indent items. I have no need for the DWM title at the top.

So each day, I just enter a new "project" at the end of the list (today I will enter April 26) and a new project in the middle of the list for today plus 7.

For projects, I would use the @ tagging method. You could follow any entry related to the "Smith Report" project with, for example, @Smith_Report. You can then use the search functionality to filter for items in that project (simply type @Smith_report into the search field.) You can also double click any "project" (in my case, the date) or any tag to do a quick filter to show the items on that date (or project) or items with that tag.

Todopaper is still new. The original "TaskPaper" for the Mac is much more capable. It allows you to save filtered searches as tabs at the top.

I originally went with taskpaper when Mark moved to (I think) Autofocus 3. This was when he began treating the tasks as if in one big list. I have early onset of arthritis and could not effectively use a notebook. I wanted something very simple that was literally a list. Back then all I wanted was new entries to appear at the end of the list and have the ability to cross out done items. Most todo programs were too much. I did not need dated entries or due dates--just a list. I thought about using a text editor but you can't easily change the font or show text as crossed out. That is when I found TaskPaper (I have an iMac, but have been booting into Windows 7 for about six months now so I switched to todopaper.)

TaskPaper (or Todopaper) works well on the earlier Autofocus systems (particularly Autofocus 4 where all you really want is a simple list.) But I love it so much that I still use it for DWM.

Wooba
I use the taskpaper iphone app mostly. They use a sync called simpletext which syncs between their iphone app, the Mac taskpaper app, and the website www.simpletext.ws. I sync between the iphone app and the simpletext.ws web site since I use Windows. From the web site I can see my task list (that I create and edit on the iphone app) and add to it like any other text file, using their formatting rules (using the tag @done for finished tasks, etc). In other words, simpletext.ws can either serve as a simple backup of your task list if you use their iphone or mac app, or it can serve as a simple stand alone online task list if you only have the iphone app and need to access your task list once and a while from any computer and don't want to buy the computer app. I also think you can use simpletext.ws by itself, since you only need a google login to access it.

It does not currently sync with the todopaper app directly, but you can copy and paste from simpletext.ws directly to the todopaper data file (which is a plain text file) and keep them in sync.
