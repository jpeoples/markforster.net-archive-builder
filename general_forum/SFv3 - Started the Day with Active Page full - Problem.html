Daouda - I am using a small notebook as you describe. Sometimes, I admit, I've been squeezing 1 or 2 extra items into the top margin of Column 2. This is probably having some detrimental effect.

Hmm, let me think through it. Is there any fundamental reason why the number of items in each Column needs to be identical? What effect would it have if you doubled the number of items in Column 2, as you propose?

Well, let's take it to an extreme: let's say I block out all but 5 lines in Column 1 (so I have space in Column 1 for only 5 tasks), and leave 25 lines in Column 2. I suppose this would have the effect that Column 2 would be able to "carry" more items forward with each page-turn. And because it COULD carry more, it WOULD carry more. This would mean I'd be drawn to start more items (because there's room in Column 2) but would not compelled to finish them as quickly. I'd quickly get Column 2 Constipation, with too many items in Column 2 that I MUST move forward or delete forever, and not enough new items in Column 1 to keep it lively and interesting. I'd basically be presented with mostly the same items in Column 2 all the time. Also, since there's not enough room for new items on each page, I'd keep adding new pages to the end (5 tasks at a time) -- I would expect this would lead to me constantly chasing the end of my list, plodding through all those same items in Column 2 all the time. I'd probably hit a wall of resistance after a while, and just quit.

Let's try the opposite thought experiment. What if I am allowed 25 items in Column 1, but only 5 items in Column 2? Column 2 would fill up VERY quickly, forcing you to dismiss the Column 1 tasks next time around. Lots of things would be forced to be dismissed before you really had any repeated exposure to them, so dismissal would lose its force. (Dismissal is supposed to take effect after repeated exposure, so you've had plenty of chances to take action but never did. If dismissal happens before that repeated exposure, then it becomes meaningless.) You'd be forced to re-enter those dismissed items pretty quickly if you really want to do them -- but they'd face dismissal again very soon. What a mess.

So, based on this thought experiment, I'd say an equal number of tasks on each page is a pretty good balance. An extra one or two items in Column 2 probably wouldn't hurt things much, except for the fact that it would lead me to take the rules less seriously overall.

While Mark's system can be extremely bottlenecked in column 2, there's a "pychological" feedback loop (via teaching the user new habits) to reduce load on column 2. However, the arrival rate of urgent tasks is not usually controllable through that loop. If urgency is externally determined (as it usually is), I see only two sane ways to cope with urgency-induced column 2 overload:

(1) make it larger (more lines) or (2) track urgent tasks outside of the system.

Mark: If the user adds lines to column 2 in response to a "chronic urgency crisis", in the long run there should be as about as many lines as before for unfinished tasks, so the only increase in things to do on every page turn is _new_ urgent tasks. Doing those preferentially is probably unavoidable anyway. (Personally, I'm back to option (2) for my system, it is more robust that way.)

I'll repeat what I said on another thread.

If everything is urgent, then nothing is urgent.

Priority is about the order we do things in. If you give the same priority to everything ("urgent") then you haven't helped at all to answer the question of what order you do things in.

It's much quicker and more efficient - all other things being equal - to complete one project before you start the next.

Whatever the perceived urgency of all your work is, it is still better to select only one or two tasks as "urgent" for Column 2.
