I think I have found my way of using AF2 now.
I have MLO (Mylife Organized) where I keep ALL my 'stuff', in a secure way, if you think GTD. It is categorized in different contexts, and I use it to find new items to put in my active list.

But I have found that Autofocus2 really makes a difference when it comes to selecting what to do during the day.
I started with a new list, not the huge archive of historic ideas.
But I use the contexts in MLO, e.g. AF2. I have then created a list with only these items, and that is the list I use daily now. New ideas are added with the rapid parser in MLO, and they automatically get the context 'AF2', so they appear on my list at once.

I am using AF2 the other way around, i.e. I have sorted my list with newst items at the top, so I read the list from top to bottom. It's much more natural for me, instead of going down to the last screen and reading backwards.

When I have done some work on an item, I just duplicate it (Ctrl-D), and tick off the active one. The new one immediately gets to the top (the bottom in original AF2). Then I go back to the top of my list, see the new item, but probably will find another task that is 'sticking out'. (If it was finsihed, I just tick it off, and it will be marked struck out in my list.)

About dismissal: Every day I look at the bottom, to find the last item i worked on. I then set a yellow flag on the last ones NOT worked on. The next day I take the decision if the flagged items should be done anything with, in that case I remove the flag, and get it done (or work for a while, and duplicate it to continue later as described above).

If I decide to really dismiss it, I just remove the AF2 tag/context from the task, and it is not on my list anymore. (But the advantage for me is that it is still in my MLO 'safe'. I know I will find it there if/when I need it later.)

I now and then do a review of my archive, and if I find something I want to work with, I set the AF2 context again, and then it is back on my working list!)

I have also added another tag/context: waiting_for. When I can't do anything more for the task unitl I get some material, or input from someone else, I add this tag. Then with MLO's automatic formatting, I get a blue flag on the line, so I easlily can see that this is in a waiting state, and I can skip it for now.

For me, I now have the perfect combination of GTD and AF2. The problem with GTD of selecting what to do is now brilliantly handled by AF2.

The problem (uneasy feeling) with AF2's dismissal is not a problem any more. I know I still have it tucked away in a safe place.
And I benefit from the advantage of really dismissing tasks (e.g. if I was too optimistic when I entered an idea on the AF2 list, or if the task for some reason could not be done just now). The active AF2 list is really active, and I have noticed that I really get some things done now.

Thank you, Mark, for sharing your ideas. I am sorry about some tweaking, i.e. resorting top-bottom. That doesn't really change the idea of AF2 at all, but it is much easier and more natural for me to use the list in that direction.
AF2 ideas has really helped me!

About MLO setup: I use Advanced filtering for a view I called Autofocus2:
contexts contains (AF2)
Sort.. by Created date / Descending

I also added an Automatic formatting rule: WF: Rule; contexts contains (Waiting) and Icon (use icon ticked) Bullet blue.
This is just to give a special indication for items where I added context=Waiting

That's it.

When one task is ready, I press spacebar, and it is struck-through.
If I want to continue on it, I first use Ctrl-d to duplicate, and it is repeated at the top of my list.

Shift-Ctrl-M is used to start Rapid task Entry. I have chosen a folder which is setup with the context AF2.
I enter as many items I want, one line for each, press Enter - and they are automatically added to (the top of) my Autofocus2 view.

Much of this requires version 3 of MLO.
