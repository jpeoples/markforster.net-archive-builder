Mark is really the person to respond but if past is prolog, he won't say too much about the "theory" of it. But I could be wrong ;-)

My understanding, based on working with the list is probably a bit simplistic but ...

- There is an active list because ... well, that is where new things are added ;-)

- The "backlog" is not actually a "backlog" but a "closed list". The rules give emphasis to it so that entries on it get priority. They will have been around longer than things on the active list and so need to get done or get gone.

- Items on the active list are de-emphasized by rule so that time is spent getting the older things done. So AF4 gives emphasis to the older items but still allows attention to be placed on the newer ones without the effect of "chasing the end of the list" that we had with previous versions.

That's about all the theory I see ;-)

The closed list is important because of the psychological effect of seeing progress. That list can only shrink, and as it does you see that you are moving forward. Simultaneously you see items that aren't moving forward and the closed list shows you that. So it's a double incentive: See the progress you're making, AND see the progress you're not making so you can rectify that.
