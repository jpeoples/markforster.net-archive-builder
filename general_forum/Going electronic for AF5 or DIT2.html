Actually I found this new DIT/AF system to be very compatible with mobile time management/calendar software such as Pocket Informant for Blackberry OS (which is what I use). The nitty gritty of my current setup is below, but in brief, what happens is

1) Every new task is automatically due 30 days (or, as I use it, 28 days) from the time I enter the said task

2) The tasks are listed chronologically: the tasks due nearer today are above the others.

3) (Now this is what I found really interesting) I can with two presses of a button set the due date of a task today so that I can highlight what I need to do; then with another few presses of a button, make its new due date one week from now.

4) Surprisingly, new tasks without any due or starting date are automatically relegated to the bottom of the heap; I therefore have a default way of listing "waiting for / something pending" tasks.

5) For my own edit of this new system, I can also with a few presses of a button to change the due date of a task /Tomorrow/, and change its priority. I can therefore set up what I think would be the most important thing to do tomorrow (reminiscent of the "Current Initiative" idea in the original DIT).


Mark, it seems you made this new system exactly for me and my phone. I thank you very heartily ^_____________^


So here are the details of my settings:

1) In Options --> Tasks
a) setup to "Start (Days from Today): 0" (This is so that the tasks show up in the "Today" view)
b) setup to "End (Days from Start): 30" (NOTE: I use 28 for this value so that the start and end days fall on the same day of the week)
c) setup to "Date, Priority, Subject"
d) check the box

2) New tasks are now automatically set as due 30 days (or in my case 28 days) from now

3) Upon doing a task, use "Menu -> Make due date today." This sets the task to the very top of the list, thus highlighting it.

4) When this task is recurrent/to be re-entered, use "Menu -> Move Task -> + Week" to move its new due date a week from now.

5) When entering a task that has a prerequisite event to happen first, remove the checks from the "Start Date" and "End Date" boxes.

I am not a paperphobe and so I use paper as my primary DWM notebook. I do like to have a copy in electronic form on the iPhone so I can do various sorts. I have played with various apps and the one I found most flexible is Toodledo. (Thanks Mel ;-) Here is how I use it, in case anyone is interested.

1) I set the default date for new tasks to be one month. When I complete a task I mark it done with one click. If it is not complete, I change the "Due Date" to "Next Week" with a two clicks.

2) There is a bulk entry ability and I used that to enter everything in my book into the program.

3) Once I had the tasks in the program, I set up some "Folders". "Folders" mean "Projects" so me. So while I may have a bunch of tasks scattered over the list about fixing the computers, I put them all in a "Computers" folder and I can look at just those when I am doing that project.

4) For those of you who like "Contexts", it has those as well.

5) You can also "Tag" tasks anyway you like. There is no practical limit to the number of tags you can use on a task.

6) I found that I can put "Events" and tasks with strict deadlines into the list simply by setting a "Start Date" to the same date as the "Due Date". They then won't appear until that date. (Unless you want them to.)

7) You can attach notes to tasks which allows putting Projects in as tasks with specific steps as notes. This can eliminate lots of clutter from the list.

8) Pocket Informant will sync with Toodledo, but I don't like that it does not give me access to Folders.

9) There is an iPhone app you can get but I have so far opted not to do that because ...

10) I can access my list via the Safari browser from the iPhone and it is excellent. If I'm out of cell phone signal access, I can always fall back on PI which I still keep synced. But that almost never happens.

11) There is an add-in for Firefox (and most other browsers). It looks exactly like the iPhone web access screen. Very nice if you are working on the computer most of the day.

12) I have not explored the gazillion other ways to access it (Twiter, eMail, etc.)

13) You can import and export in CSV and Text formats. There is a one button export to Excel, for example, if you want to keep other kinds of information, such as statistics.

14) You can share tasks with others in a number of ways.

15) It has the ability to print one of those Origami pocket books. You know, where you cut and fold it up to make a letter size sheet to be the size of a postage stamp? ;-)

There are other features such as priority and such. It even calculates an "Importance" number which uses a formula considering priority, number of days until due, etc.

All in all, the combination of Toodledo and the iPhone is almost perfect. Of course, I STILL enter my information into my paper book FIRST. It is still faster to do it that way. At least for me. And I am completely sure that my paper won't suddenly start costing me big bucks because the business model was not proving profitable ;-)

Oh, and it is FREE! Unless you want some of the Cadillac features, such as sub-tasks and file storage.

@Mel & @Mike,
In Toodledo "Due by" is the typical Due Date. The task must be completed *by* this date. "Due on" simply means that the tasks is like an appointment. It is due *on* that date. The "Optionally On" means that the task will be removed from the list if not done by the date and if it is recurring it will be moved forward. For more info,

http://www.toodledo.com/info/help.php?sel=42
