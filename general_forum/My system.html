Such a flurry of activity this week! That can only lead to great ideas becoming even better ideas. Here's my idea, which I've been holding tight for three months because I wanted first to be sure it works.

The Fundamental Idea:
The system uses an AutoFocus type framework to encourage these work habits:
- Focus only on a few things you've started. Stay focused.
- When something new comes up, put it off.
- When you want to work on something new, first work on something old, so nothing gets forgotten.

It's hard work writing things clearly. I hope I succeeded.

1. Start with two pages. Over time, more pages will be active. All pages are closed lists except the last two.

2. The second last page is open to new tasks. Whenever a new task comes to mind, write it here.

3. The last page is called Focus. Whenever you work on a task, rewrite it to the Focus page.

4. At any time, you may start a new Focus page.
• On the old Focus page , draw a heavy line below the task list.
• The old Focus is now ready for new tasks,
• The new Focus page is ready for worked-on tasks.
• Continue working where you left off.


(To work a page, go through tasks in order. Work on any task as you wish, rewriting the task to the Focus page. Resume scanning when done.)

1. Work the Focus page. Repeat as often as you can. [This is the whole point of this system.]

2. Work the oldest page. You *must* activate or dismiss at least one task. If you don't, keep scanning until you find something to activate or dismiss. Discard the page when no tasks remain.

3. Work the other pages in order, then return to the Focus page.

Yes, the New Page rule is incomplete. Ideally it would tell you when to make a new page, but I haven't figured out when is best, so I left it unspecified. Flexibility may be an advantage though. I have tons of possibilities you could try.

AF4 mode: When the first page clears, I tried this for a long time, and it really pressures you into dismissing. But sometimes the first page doesn't clear fast enough, and sometimes it clears too quickly.

DIT mode: near the end of your day, create a new page. Scan through the pages selecting the important tasks you want to achieve tomorrow, and prepare for each of them, thus shifting them to the active page.

"No wasted paper": when the Focus page is3/4 full, or the previous page is full.

Refocusing: A trick that fits the flow of the system: If you pick up too many tasks from going around, just start a new page and only reactivate a few tasks from the current focus. You don't have to go through everything again.

New context: If you are switching contexts, start a new page, and pick up tasks that will be useful in the new context.

Project-zoom: Take a project page or checklist and copy tasks from that into a new Focus page.


Re the other novel idea: I don't recall that discussion. But here only the first page demands something be chosen and worked on. If you had to chose one thing on every page, you might get too many things.

Thanks for the encouragement Erik! Following the revised page configuration, the focus page has new tasks on the left and active tasks ( what I have been working on ) on the right, and you would loop through tasks on the right side only. Here's how that plays out in practice:

Effectively there are two phases in the system:
1. Picking (and starting) tasks from the full list.
2. Working recently picked tasks.
Suppose I have no tasks on the right (focus list). Then I will scan through all the pages, maybe picking 10 along the way and doing a little work on each as I go.

As I return to focus, I repeatedly pick among those 10 only. Likely I will stick to just a few (3) and get those few progressed, and later switch to a few others, and the rest don't really need more work at this time.

New stuff will be written on the left, but I should not work on it yet. If it's truly necessary I must switch to the picking phase and pick more tasks, old and new. The effect is to discourage distraction because normally I don't want to pay the price to start new tasks before I finish the current ones

One useful work pattern is to start a new page shortly before leaving work. Then I scan through looking for what I want to do tomorrow and prepare each. Then my focus list is exactly my DIT plan, and I have each task ready.

Likewise, if I'm going on errands I can start a new page, collect tasks useable on the road, and then get to them.

These are the current ideas. I'm open to refinement.

My System of focusing tasks also works with DWM2:

1. Each day write a new date as per DWM2 rules.
2. Enter tasks exactly according to DWM2 rules.
3. Dismiss tasks exactly according to DWM2 rules.
4. Scan through all tasks starting with the oldest. Work on any, and rewrite with a black dot.
5. Now Focus exclusively on today's tasks with a black dot.
6. When you want to work on something else (including today's items without a black dot), go to 4.

I follow the above process except for two things:
- Instead of dots I use two columns: left = new, right = re-entered.
- I don't dismiss by date, but by lack of activity.

You sum it well. Except the "absorbed into the main list" doesn't happen every time 'round. Also, in later posts I suggested that rather than 2 pages you have 2 columns: left is unactioned tasks, right is actioned tasks.

The systems are similar in these factors: 2 columns, re-entry on right, page by page working, focusing on one page for a while, and a general autofocusy style (record everything, go round and round, dismiss at appropriate times).

The primary mechanical differences are only three: rewrite to last half-page vs. rewrite to current half-page, and loop each page vs loop last half-page, and the dismissal rules.

The practical differences are though VERY significant:
- Look through everything quickly, vs. look at one page
- Focus only on a few things you've started, vs. focus on one page at a time
- When something new comes up, put it off vs. do it now if you want
- Work old pages before new vs. every page treated equally.

The difference is because Super AF comes out of AF1, whereas my system evolved out of inverting the AF4 open/closed list [which doesn't work], and DWM's separate treatment of new and re-entered tasks.

(I never got into AF1 because it didn't make sense - in my context - to work everything equally. Some things need to get done now. The rest need to get done, but not now. Running through AF1 meant either dismissing many pages of stuff, or else taking too long to get to the few things that need doing now.)

Your first guess was correct. I insist on working on things to activate tasks. This discourages grabbing too many things, and encourages preparing to actually get stuff done.

To achieve "Quickly", I keep my list short, and pick very few things. Usually this takes under an hour. If something is "right-now urgent", I'll go do it. More usually it's "same-day urgent", and I'll get to it shortly.

E.g., Phone rings, I answer it. Talk to customer. They have a problem with a file. Need it fixed today. I write it on my list as a new thing. I continue what I was already doing, wrap that up [done for now], then go get that new thing, following the rules.

The biggest factor to avoid interruptions is that I anticipate what will come up. I know the things going on around here, and generally those things are on my Focus list.

E.g., I see an item languishing. I know it's important, so I ask somebody working on it, how it's progressing. Note response, item is activated. Now I might use that active item plan what's next, and the interruption "I'm done" just fits my flow.
